Dois Tipos de Armazenamento

LocalStorage

  • guarda pares chave = valor
  • ideal para preferências e dados pequenos
  • muito fácil de usar

IndexedDB

  • funciona como banco de dados do navegador
  • guarda muitos registros e objetos complexos
  • melhor para aplicações maiores
Nesta aula, vou tratar o que você chamou de dbstorage como IndexedDB, que é a solução de banco local do navegador.

Passo 1: Primeiro Contato com localStorage

localStorage.setItem('nome', 'Ana');

const nomeSalvo = localStorage.getItem('nome');
console.log(nomeSalvo);

setItem

guarda um valor usando uma chave.

getItem

lê o valor salvo naquela chave.

Ponto importante

o localStorage guarda texto.

Exemplo 1: Salvar o Nome do Usuário

<input id="nome" placeholder="Digite seu nome">
<button onclick="salvarNome()">Salvar</button>
<div id="saida"></div>

function salvarNome() {
  const nome = document.getElementById('nome').value;
  localStorage.setItem('nomeUsuario', nome);
  document.getElementById('saida').innerText = 'Nome salvo!';
}

Aqui o aluno digita, clica no botão e já vê um retorno imediato na tela.

Exemplo 2: Recuperar ao Recarregar a Página

function mostrarNomeSalvo() {
  const nome = localStorage.getItem('nomeUsuario');

  if (nome) {
    document.getElementById('saida').innerText =
      `Bem-vindo de volta, ${nome}!`;
  }
}

window.addEventListener('load', mostrarNomeSalvo);

O que o aluno percebe

mesmo recarregando a página, o dado continua ali.

Ideia central

localStorage persiste depois do refresh e até depois de fechar o navegador.

Exemplo 3: Lista de Tarefas com localStorage

function adicionarTarefa() {
  const texto = document.getElementById('tarefa').value;
  const tarefas = JSON.parse(localStorage.getItem('tarefas')) || [];

  tarefas.push(texto);
  localStorage.setItem('tarefas', JSON.stringify(tarefas));

  listarTarefas();
}

function listarTarefas() {
  const tarefas = JSON.parse(localStorage.getItem('tarefas')) || [];
  document.getElementById('saida').innerHTML =
    tarefas.map(t => `<li>${t}</li>`).join('');
}
Para guardar arrays e objetos, usamos JSON.stringify() ao salvar e JSON.parse() ao ler.

Quando localStorage é uma Boa Escolha?

Ótimo para

tema escuro, nome do usuário, idioma, carrinho simples, filtros de tela.

Não é ideal para

muitos registros, buscas complexas, relacionamentos e grandes volumes de dados.

Resumo

simples, rápido e ótimo para pequenas persistências.

Regra prática: se cabe como algumas chaves e poucos dados, comece com localStorage.

Entrando no IndexedDB

O IndexedDB é o banco de dados embutido no navegador.

  • guarda muitos registros
  • aceita objetos completos
  • trabalha com transações
  • é assíncrono

Pense assim

localStorage parece uma caixinha de rótulos; IndexedDB parece uma mini base local.

Uso comum

catálogos, rascunhos offline, dados de apps e registros maiores.

Passo 1: Abrir o Banco

let db;

const request = indexedDB.open('LojaDB', 1);

request.onsuccess = function (event) {
  db = event.target.result;
  console.log('Banco aberto com sucesso');
};

LojaDB

é o nome do banco.

1

é a versão do banco.

onsuccess

é chamado quando a abertura funciona.

Passo 2: Criar a Estrutura

request.onupgradeneeded = function (event) {
  db = event.target.result;

  db.createObjectStore('produtos', {
    keyPath: 'id',
    autoIncrement: true
  });
};

onupgradeneeded

roda quando o banco é criado pela primeira vez ou quando a versão muda.

objectStore

funciona como uma “tabela” dentro do banco.

Passo 3: Inserir um Registro

function adicionarProduto() {
  const tx = db.transaction('produtos', 'readwrite');
  const store = tx.objectStore('produtos');

  store.add({
    nome: 'Mouse',
    preco: 59.90
  });
}

transaction

abre uma transação para leitura ou escrita.

readwrite

permite gravar dados.

store.add()

insere um novo objeto na store.

Passo 4: Listar os Dados

function listarProdutos() {
  const tx = db.transaction('produtos', 'readonly');
  const store = tx.objectStore('produtos');
  const request = store.getAll();

  request.onsuccess = function () {
    const produtos = request.result;

    document.getElementById('saida').innerHTML =
      produtos.map(p => `<li>${p.nome} - R$ ${p.preco}</li>`).join('');
  };
}

Esse é o momento em que a turma enxerga o banco local funcionando de verdade: grava e depois lista.

Exemplo Prático Completo

<input id="produto" placeholder="Nome do produto">
<button onclick="salvarProduto()">Salvar</button>
<ul id="saida"></ul>

function salvarProduto() {
  const nome = document.getElementById('produto').value;
  const tx = db.transaction('produtos', 'readwrite');
  const store = tx.objectStore('produtos');

  store.add({ nome: nome });
  tx.oncomplete = listarProdutos;
}
Em sala, esse exemplo costuma funcionar bem porque o aluno vê a diferença entre guardar um valor isolado e guardar vários registros.

LocalStorage x IndexedDB

Critério LocalStorage IndexedDB
Modelo chave e valor banco com stores e registros
Tipo de uso simples e rápido mais robusto
Tamanho e volume pequeno muito maior
Dados complexos precisa converter para JSON lida melhor com objetos
Curva de aprendizado baixa maior

Erros Comuns

Salvar objeto direto no localStorage

sem JSON.stringify(), o valor não fica no formato esperado.

Esquecer o JSON.parse()

o dado volta como texto, não como array ou objeto.

Usar o IndexedDB antes de abrir o banco

se db ainda não foi inicializado, as funções falham.

Confundir tabela com objectStore

no IndexedDB falamos em objectStore, não exatamente em tabela SQL.

Exercício Guiado

  • Crie um formulário para salvar a cor favorita com localStorage.
  • Ao recarregar a página, mostre a cor salva e aplique no fundo.
  • Depois, crie uma objectStore chamada anotacoes no IndexedDB.
  • Permita inserir várias anotações e listá-las na tela.
  • Pergunta final para a turma: qual parte faria mais sentido guardar em cada tecnologia?
