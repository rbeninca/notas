A força dessa sequência é que o padrão do código quase não muda: mudam a URL, o tipo de dado e o campo que queremos mostrar.

Antes de tudo: rode em servidor local

Para começar com texto.txt e dados.json, o ideal é abrir a pasta com Live Server ou com um servidor simples.

python3 -m http.server

Depois acesse o endereço mostrado no terminal, como http://localhost:8000.

  • index.html com o botão e a função JavaScript
  • texto.txt para o primeiro teste
  • dados.json para o segundo teste
  • Um <div id="conteudo"> para exibir o resultado
Se o HTML for aberto direto como file://, o fetch() pode falhar. Em aula, vale muito a pena já mostrar isso cedo.

Passo 1: Buscar um arquivo texto

<button onclick="carregarConteudo()">Carregar Conteúdo</button>
<div id="conteudo"></div>

<script>
function carregarConteudo() {
  fetch('texto.txt')
    .then(response => response.text())
    .then(texto => {
      document.querySelector('#conteudo').textContent = texto;
    });
}
</script>

O aluno vê

Clica no botão e o conteúdo de texto.txt aparece no div.

Ideia principal

A URL ainda é só um arquivo da própria pasta.

Ponto-chave

Como é texto simples, usamos response.text().

Passo 2: Trocar texto por JSON

Arquivo dados.json

{
  "nome": "Ana",
  "curso": "ADS",
  "periodo": 3
}

Leitura no JavaScript

function carregarAluno() {
  fetch('dados.json')
    .then(response => response.json())
    .then(aluno => {
      document.querySelector('#conteudo').textContent =
        `${aluno.nome} - ${aluno.curso}`;
    });
}

Aqui os alunos percebem uma mudança muito importante: o dado não é mais uma string; agora ele vira um objeto.

O que mudou de verdade?

Continua igual

fetch(...) inicia a requisição e devolve uma Promise.

Muda aqui

response.text() lê texto. response.json() converte para objeto JavaScript.

Objetivo final

No final, quase sempre queremos pegar um valor e colocar na tela.

Modelo mental: buscar → ler a resposta → extrair o campo desejado → atualizar o DOM

Reescrevendo com async e await

async function carregarAluno() {
  const response = await fetch('dados.json');
  const aluno = await response.json();

  document.querySelector('#conteudo').textContent =
    `${aluno.nome} - ${aluno.curso}`;
}
  • async marca a função assíncrona
  • await fetch(...) espera a chegada da resposta
  • await response.json() espera a conversão do corpo
  • A leitura fica mais próxima da ordem em que explicamos no quadro
await não “trava a página inteira”; ele pausa só aquela função até o resultado chegar.

API Externa: Chuck Norris

async function carregarPiada() {
  const url = 'https://api.chucknorris.io/jokes/random';
  const response = await fetch(url);
  const dados = await response.json();
  const conteudo = document.getElementById('conteudo');

  conteudo.innerText = dados.value;
}

URL pronta

/jokes/random já devolve uma piada aleatória em JSON.

Campo importante

A frase que queremos mostrar está em dados.value.

Por que funciona bem em aula

É rápido, divertido e o resultado aparece imediatamente na interface.

Chuck Norris: extensões naturais

Outra URL útil

https://api.chucknorris.io/jokes/categories devolve a lista de categorias.

Piada por categoria

/jokes/random?category=dev permite brincar com filtros.

Busca textual

/jokes/search?query=car já introduz a ideia de parâmetros na URL.

Esse momento é ótimo para mostrar que a estrutura do código é a mesma; o que muda é a URL e o campo do JSON que vamos usar.

Passo 4: OMDb API

A OMDb já aproxima muito mais de um cenário real de aplicação:

  • exige apikey
  • permite buscar por t (título) ou s (pesquisa)
  • aceita filtros como type e y

Exemplo de base

https://www.omdbapi.com/?apikey=SUA_CHAVE&t=Batman

Ponto prático

Sem chave válida, a API responde com erro no JSON.

Na prática de aula, isso ajuda a mostrar que consumir API real quase sempre envolve mais um detalhe de configuração. A página de chave da OMDb informa plano gratuito com limite diário de 1.000 requisições.

OMDb: buscar um filme por título

async function buscarFilme() {
  const titulo = document.querySelector('#titulo').value;
  const url = `https://www.omdbapi.com/?apikey=SUA_CHAVE&t=${encodeURIComponent(titulo)}`;

  const response = await fetch(url);
  const filme = await response.json();

  if (filme.Response === 'False') {
    document.querySelector('#conteudo').textContent = filme.Error;
    return;
  }

  document.querySelector('#conteudo').innerHTML =
    `<h3>${filme.Title} (${filme.Year})</h3>
     <p>${filme.Genre}</p>`;
}

Campo do título

Agora a URL depende do que o usuário digitou.

Resposta da API

O JSON já traz campos úteis como Title, Year e Genre.

Ganho didático

Os alunos percebem a passagem do “arquivo local” para “consulta montada dinamicamente”.

OMDb: pesquisar vários filmes

async function pesquisarFilmes() {
  const termo = document.querySelector('#busca').value;
  const url = `https://www.omdbapi.com/?apikey=SUA_CHAVE&s=${encodeURIComponent(termo)}`;

  const dados = await (await fetch(url)).json();

  if (dados.Response === 'False') {
    document.querySelector('#conteudo').textContent = dados.Error;
    return;
  }

  const itens = dados.Search
    .map(filme => `<li>${filme.Title} (${filme.Year})</li>`)
    .join('');

  document.querySelector('#conteudo').innerHTML = `<ul>${itens}</ul>`;
}

Esse exemplo é muito bom para conectar fetch com manipulação de lista e geração de HTML no DOM.

Erros práticos nessa sequência

Abrir no file://

O exemplo de texto.txt pode falhar se a página não estiver sendo servida por HTTP.

Trocar text() por json()

Quando o arquivo é JSON, esquecer essa troca impede o acesso aos campos como nome ou curso.

Esquecer o segundo await

response.json() também é assíncrono; ele não devolve o objeto imediatamente.

Na OMDb, confiar só no HTTP

A API pode responder com erro dentro do próprio JSON, como quando a apikey é inválida.

Roteiro prático com a turma

  • Comece com texto.txt e peça para eles preverem o que aparecerá no div.
  • Troque só uma linha: text() por json(), e mostre o salto para objeto JavaScript.
  • Reescreva com async/await mantendo a mesma lógica.
  • Mude apenas a URL para a Chuck Norris API e pergunte: “qual campo do JSON interessa agora?”
  • Finalize com a OMDb, mostrando que aplicações reais costumam pedir apikey e validação extra.

Resumo mental

1. Buscar

fetch(url)

2. Ler

text() ou json()

3. Extrair

pegar o campo certo

4. Mostrar

atualizar o DOM

5. Repetir

agora com outra URL

async function buscarDados(url) {
  const response = await fetch(url);
  return await response.json();
}
O padrão-base é quase sempre o mesmo. Em aula, isso ajuda os alunos a perceberem que consumir API não é “aprender outro mundo”; é repetir uma estrutura conhecida.
